# Cassandra Start Guide

> ATTENTION: This driver is supported on v0.21.9 or older only!

## 1. Connections

Connection example:
```json
{
  "name": "Cassandra",
  "server": "localhost",
  "driver": "CQL",
  "port": 9042,
  "database": "my_keyspace", // Specifies a keyspace instead of a database. This field can be omitted.
  "username": "sa",
  "askForPassword": false,
  "password": "root(!)Password",
  "connectionTimeout": 15,
}
```

### 1.1 Multiple Hosts

You can pass a list of hosts by separating them with a comma (`,`), instead of a single server:

```json
{
  "name": "Cassandra",
  "server": "host1.example.com,host2.example.com,2001:0db8:85a3::1",
  "driver": "CQL",
  "port": 9042,
  "database": "my_keyspace",
  "username": "sa",
  "askForPassword": false,
  "password": "root(!)Password",
  "connectionTimeout": 15,
}
```

### 1.2 Specific Options

Cassandra driver specific options can be passed using `cqlOptions` settings.

```json
{
  "name": "Cassandra",
  "server": "localhost",
  "driver": "CQL",
  "port": 9042,
  "database": "my_keyspace",
  "username": "sa",
  "askForPassword": false,
  "password": "root(!)Password",
  "connectionTimeout": 15,
  "cqlOptions": {
    ... // options
  }
}
```

You can use any options defined in https://docs.datastax.com/en/developer/nodejs-driver/4.1/api/type.ClientOptions/ in `cqlOptions`.
